Client Site - On Different Domain

If the page in the iframe fails to set a cookie it will redirect the top window to a page on the same domain as the iframe so that a cookie can then be set

Once the cookie is set, the user will be redirected back to this same parent loading the same child.

This does not rely on browser detection (horrible), but attempts to set a cookie and if that fails, redirects the user.

If the user returns and a cookie was not set, inform the user and assume the user has disabled cookies.

Notes

  • This site (index.html) and the iframe source (iframe.html) must be on separate domains to demonstrate the issue with setting third party cookies for specific browsers.
